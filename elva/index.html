02 — Where to move next

Three pillars.

Motion with meaning

Transitions and state changes that guide rather than distract. Revolut-level intentionality — purposeful, not decorative.

Interaction design

Human-first hierarchy

People are not case numbers. Lead with the person — their name, their progress, their next step — not a generic task list.

Information architecture

Scalable by design

Every decision becomes a token — not a one-off. A system any engineer can extend without breaking the craft already built in.

02 — Where to move next

Four principles.

01

Show, don't list

Replace flat text with visual progress. The journey should be felt, not just read.

02

Calm complexity

Relocation is stressful. The UI should reduce anxiety, not mirror it.

03

Earn every moment

Micro-interactions only where they add meaning — never for decoration.

04

One language

A token-based system so every new screen inherits the craft automatically.

05 — Next steps

Let's build this
together.

This is a starting point, not a final word. The direction shows what's possible — and opens a real conversation about how to get there. Everything stays within Elva's existing brand.

1

Design system foundations

Token-based colour, spacing, radius, and motion — a shared language that scales with the product.

2

Core screen redesigns

Timeline, tasks, home — the 3–4 screens employees touch most, elevated with the new direction.

3

Interaction & motion library

The micro-animations and state changes that make the product feel alive — documented and reusable.

4

Component library

Figma components built on the system — ship confidently without breaking the craft.
